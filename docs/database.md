# Схема базы данных

В текущей версии приложения полноценная SQL база данных не используется ввиду легкости данных. Хранение реализовано через **SharedPreferences** в формате **JSON-массива**.

## Структура JSON объекта (MapPoint)
```json
{
  "x": 0.5,           // Координата X (Float, от 0.0 до 1.0)
  "y": 0.5,           // Координата Y (Float, от 0.0 до 1.0)
  "title": "String",  // Название точки
  "description": "String" // Подробное описание
}
```

## SQL файл
Так как SQLite не используется, SQL-файл отсутствует. Вся работа с данными происходит в методах `savePointsToPrefs` и `loadPointsFromPrefs` в файле `MainActivity.kt`.
